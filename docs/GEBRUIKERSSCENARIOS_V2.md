# Projojo Gebruikersscenario's v2

> Geherstructureerde visie op het platform: matching van vraag en aanbod in het onderwijs-praktijk ecosysteem.

---

## De Kernvraag: Voor wie en wat lossen we op?

Projojo is een **matchingsplatform** dat vraag en aanbod in het onderwijs-praktijk ecosysteem bij elkaar brengt.

```
VRAAGKANT                              AANBODKANT
┌──────────────────────┐              ┌──────────────────────┐
│                      │              │                      │
│  ORGANISATIES        │              │  STUDENTEN           │
│  ├─ Praktijkvragen   │              │  ├─ Skills           │
│  ├─ Capaciteitsnood  │              │  ├─ Beschikbaarheid  │
│  └─ Expertise nodig  │              │  └─ Interesses       │
│                      │              │                      │
│  DOCENTEN            │   MATCHING   │                      │
│  ├─ Leerdoelen       │◄────────────►│                      │
│  ├─ Curriculumeisen  │              │                      │
│  └─ Studentplaatsen  │              │                      │
│                      │              │                      │
└──────────────────────┘              └──────────────────────┘
```

**Het platform lost op:**
1. Organisaties vinden gekwalificeerde studenten voor praktijkvragen
2. Studenten vinden relevante praktijkervaring die past bij hun opleiding
3. Docenten kunnen curriculum koppelen aan echte praktijkprojecten
4. Iedereen bouwt aan een verifieerbaar portfolio

## Types Opdrachten

Niet elke opdracht is hetzelfde. Het platform ondersteunt verschillende types engagement:

### Classificatie-eigenschappen

| Eigenschap | Opties | Invloed op matching |
|------------|--------|---------------------|
| **Type engagement** | Stage / Afstuderen / Minor / Losse klus / Betaald | Bepaalt formele eisen |
| **Omvang** | Uren per week (4/8/16/32/40) | Filtert op beschikbaarheid |
| **Duur** | Weken (1-4 / 5-12 / 13-26 / 26+) | Past bij semester/periode |
| **Samenstelling** | Individueel / Duo / Groep (3-5) / Team (6+) | Bepaalt aantal posities |
| **Niveau** | HBO / WO / MBO | Voorkomt mismatch |
| **Studiejaar** | Jaar 1 / 2 / 3 / 4 / Master | Complexiteit passend |
| **Werkvorm** | Op locatie / Remote / Hybride | Praktische haalbaarheid |
| **Beloning** | Studiepunten / Certificaat / Betaald / Ervaring | Motivatie/verwachting |

### Voorbeelden per Type

**Stage (traditioneel)**
- Omvang: 32-40 uur/week
- Duur: 20 weken
- Individueel
- Studiepunten gekoppeld
- Op locatie

**Afstudeeropdracht**
- Omvang: 24-40 uur/week
- Duur: 16-24 weken
- Individueel of duo
- Studiepunten + begeleiding docent
- Hybride

**Minor-project**
- Omvang: 8-16 uur/week
- Duur: 10-20 weken
- Groep (3-5 studenten)
- Studiepunten via docent
- Flexibel

**Losse klus (extracurriculair)**
- Omvang: 4-8 uur/week
- Duur: 2-8 weken
- Individueel
- Ervaring + portfolio
- Remote mogelijk

**Betaalde opdracht**
- Omvang: Variabel
- Duur: Projectgebonden
- Individueel (specialist)
- Betaald (uurtarief/fixed)
- Afspraken per case

## Skills vs Interesses/Thema's

Een belangrijk onderscheid in het platform:

| Aspect | Skills | Interesse/Thema |
|--------|--------|-----------------|
| **Definitie** | Concrete vaardigheden, kennis | Onderwerpen die je interesseren |
| **Bron** | Opleiding, cursussen, ervaring | Persoonlijke motivatie, nieuwsgierigheid |
| **Voorbeelden** | Python, Projectmanagement, GIS | Duurzaamheid, AI, Voedselzekerheid |
| **Wie beheert** | Student voegt toe, Docent kan verifiëren | Student kiest, Organisaties kunnen voorstellen |
| **Matching** | **Hard filter**: vereist voor taak | **Soft filter**: voorkeur, niet vereist |
| **Verificatie** | Kan endorsed worden door organisatie | Geen verificatie nodig |

### Hoe werkt dit in de praktijk?

**Voor studenten:**
- Skills komen primair uit opleiding (automatisch of handmatig)
- Interesses/thema's kies je zelf (max 5)
- Bij projecten zie je: "Vereiste skills ✓" en "Past bij jouw interesses"

**Voor organisaties:**
- Bij project aanmaken: selecteer vereiste skills
- Koppel max 3 thema's aan je project
- Nieuwe thema's kunnen voorgesteld worden (docent keurt goed)

**Voor docenten:**
- Beheer de skills-bibliotheek (goedkeuren nieuwe skills)
- Beheer thema's/interesse-gebieden
- Kan skills van studenten endorsen/verifiëren

## Perspectief: Docent als Vragende Partij

**Nieuw:** Docenten kunnen niet alleen toezicht houden, maar ook actief projecten aanvragen.

### Scenario A: Projecten voor een vak/minor

```
┌─────────────────────────────────────────────────────────────┐
│ HENK - Docent minor "Duurzame Landbouw"                     │
│                                                             │
│ Situatie:                                                   │
│ - 20 studenten in de minor                                  │
│ - Zoekt 5-6 praktijkprojecten                              │
│ - Periode: September - Januari                              │
│ - Elk project voor 3-4 studenten                           │
└─────────────────────────────────────────────────────────────┘

FLOW:
1. Docent maakt "Opdrachtvraag" aan:
   ├─ Koppelt aan vak/minor
   ├─ Definieert leerdoelen
   ├─ Stelt eisen aan organisaties (begeleiding, etc.)
   └─ Geeft periode en gewenste omvang aan

2. Opdrachtvraag wordt zichtbaar voor organisaties:
   ├─ "Minor Duurzame Landbouw zoekt projecten"
   ├─ Organisaties kunnen reageren met projectvoorstel
   └─ Meerdere organisaties kunnen matchen

3. Docent reviewt en keurt goed:
   ├─ Beoordeelt of project past bij leerdoelen
   ├─ Keurt projecten goed of af
   └─ Goedgekeurde projecten worden "curriculum-linked"

4. Studenten kiezen:
   ├─ Zien goedgekeurde projecten met "Minor X" badge
   ├─ Kunnen voorkeuren aangeven
   └─ Docent verdeelt/matcht
```

### Scenario B: Specifieke onderzoeksvraag

```
┌─────────────────────────────────────────────────────────────┐
│ HENK - Docent met onderzoeksidee                            │
│                                                             │
│ Situatie:                                                   │
│ - Wil onderzoek naar "impact IoT op kleine boerderijen"    │
│ - Zoekt organisatie die als case wil dienen                │
│ - Geschikt voor 1-2 afstudeerders                          │
└─────────────────────────────────────────────────────────────┘

FLOW:
1. Docent plaatst onderzoeksvraag:
   ├─ Beschrijft onderzoeksthema
   ├─ Definieert wat organisatie moet bieden (data, toegang)
   └─ Geeft aan voor welk type student

2. Organisaties tonen interesse:
   ├─ "Wij willen graag meewerken"
   ├─ Beschrijven hun situatie/case
   └─ Geven aan wat ze kunnen bieden

3. Match wordt gemaakt:
   ├─ Docent kiest passende organisatie(s)
   └─ Project wordt concreet gemaakt

4. Student wordt gezocht:
   ├─ Project wordt gepubliceerd
   ├─ Studenten kunnen solliciteren
   └─ Docent + organisatie selecteren samen
```

### Scenario C: Koppelaar van bestaande projecten

```
┌─────────────────────────────────────────────────────────────┐
│ HENK - Docent ziet interessante projecten                   │
│                                                             │
│ Situatie:                                                   │
│ - Bladert door bestaande projecten op platform             │
│ - Ziet project dat perfect past bij zijn vak               │
│ - Wil dit "adopteren" voor curriculum                      │
└─────────────────────────────────────────────────────────────┘

FLOW:
1. Docent markeert project:
   ├─ "Geschikt voor [Vak X]"
   ├─ Koppelt studiepunten
   └─ Voegt eventueel extra leerdoelen toe

2. Voor studenten:
   ├─ Project krijgt badge "Aanbevolen voor [Vak X]"
   ├─ Telt mee voor studiepunten als ze het doen
   └─ Docent wordt CC bij aanmelding

3. Voor organisatie:
   ├─ Ziet dat project curriculum-linked is
   ├─ Weet dat docent meekijkt
   └─ Extra kwaliteitsgarantie
```

## Roadmap: Gefaseerde Implementatie

### Fase 1: MVP / Live Test

**Doel:** Basisplatform draaien met uitgebreide opdracht-eigenschappen

- [ ] Opdracht-classificatie toevoegen (type, omvang, niveau, samenstelling)
- [ ] Basis student-funnel ("Wat zoek je?" bij onboarding)
- [ ] Docent kan bestaande projecten "adopteren" voor curriculum
- [ ] Thema's/interesses naast skills

### Fase 2: Funnels & Docent-features

**Doel:** Betere matching door begeleide vraagformulering

- [ ] Volledige organisatie-funnel (wizard bij project aanmaken)
- [ ] Geavanceerde student-funnel met beschikbaarheid
- [ ] Docent kan "opdrachtvraag" plaatsen
- [ ] Studiepunten koppeling
- [ ] Notificaties bij matches

### Fase 3: Geavanceerde Features

**Doel:** Ecosysteem volledig operationeel

- [ ] Betaalde opdrachten integratie
- [ ] Matching-algoritme verbetering
- [ ] Team-aanmeldingen (groep solliciteert samen)
- [ ] Portfolio/reviews volledig werkend
- [ ] Externe integraties (edubadges, LMS, LinkedIn)

---

## Toekomstige Rollen (Fase 3)

Naast de huidige rollen (Student, Organisatie, Docent) zijn er toekomstige rollen gepland:

### Burger/Ervaringsdeskundige
- Niet-student, niet-professional
- Kan expertise inbrengen vanuit levenserfaring
- Voorbeeld: iemand met visuele beperking helpt bij accessibility project
- Registratie apart van eduID

### Hub/Holding/Netwerk
- Overkoepelende organisatie
- Beheert meerdere projecten/dochterorganisaties
- Faciliteert cross-company samenwerking
- Gecombineerde statistieken en rapportage

### Admin (apart van Docent)
- Platform-breed beheer
- Gebruikersrollen toewijzen
- Analytics en monitoring
- Audit logging

### Gap Analyse

**Volledig geimplementeerd (MVP):**
- Basis dashboards (student, supervisor)
- Projecten zoeken en filteren
- Kaart/geo visualisatie
- Basis authenticatie en profielen

**Deels geimplementeerd:**
- Portfolio (schema aanwezig, UI mist)
- Docent rechten (kan zien, niet beheren)

**Grote gaps (Fase 2):**
- Multi-supervisor per bedrijf
- Reviews en endorsements
- Docent project-adoptie
- Betaald werk / werkstudent
- Funnel flows

**Toekomst (Fase 3):**
- Burger/Expert rol
- Cross-company projecten
- Hub/Holding organisaties
- Admin dashboard

## Samenvatting: Wat Lost Projojo Op?

| Voor | Probleem | Oplossing |
|------|----------|-----------|
| **Student** | "Waar vind ik praktijkervaring die past bij mijn skills en beschikbaarheid?" | Gefilterde matches, gepersonaliseerde aanbevelingen |
